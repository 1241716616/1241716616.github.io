# 树

数组的优缺点：

优点：可以利用二分查找。
缺点：插入和删除都比较慢，如果数据量比较大，需要挪动大量的数据。

链表的优缺点：
优点：链表插入和删除的时候非常快。
缺点：不能进行二分查找，因为链表它没有下标。

树这种数据结构结合就数组的优点和链表的优点。

# 树的相关术语

  • 结点
  • 父结点
  • 子节点
  • 兄弟结点
  • 叶子结点（终端结点）：无子结点的结点
  • 度：一个结点的子树个数
  • 树的高度：层数

# 二叉树

二叉树是 n (n>=0) 个结点的有限集合，该集合或者为空集（称为空二叉树），或者由一个根结点和两棵互不相交的、分别称为根结点的左子树和右子树组成。

二叉树有以下特点：
（1）每个结点最多有两颗子树
（2）左子树和右子树是有顺序的，次序不能任意颠倒（小的数放在左边，大的数放在右边）；即使树中某结点只有一棵子树，也要区分它是左子树还是右子树。

# 二叉树的遍历

  • 二叉树的遍历：是指从二叉树的根结点出发，按照某种次序依次访问二叉树中的所有结点，使得每个结点被访问一次，且仅被访问一次。

二叉树的遍历共有四种：

  • 中序遍历：左子树 - 节点 - 右子树；针对上述模型最后就是 A/B * C-D+E。
  • 前序遍历：节点 - 左子树 - 右子树；最后结果为：+-*/ABCDE。
  • 后序遍历：左子树 - 右子树 - 节点；最后结果为：AB/C*D-E+。
  • 层序遍历：一层一层遍历；最后结果为：+-E*D/CAB。

# 二叉查找树

又称二叉排序树（Binary Sort Tree），亦称二叉搜索树。

特点：

  1. 每一个元素有一个键值，且不允许重复；
  2. 左子树的键值都小于根结节的键值；
  3. 右子树的键值都大于根结节的键值；
  4. 左右子树也都是二叉查找树，即也满足上述性质。

缺点 是有出现链表的可能性，则不能进行二分查找，即没有一个动态平衡的功能。

代码实现二叉查找树：
若根结点的键值等于查找的值，成功。否则，若小于根结点的键值，递归查左子树。若大于根结点的键值，递归查右子树。若子树为空，查找不成功。
