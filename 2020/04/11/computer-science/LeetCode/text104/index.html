# 解法二

  • 解题思路：使用哈希函数。


总结： 哈希函数的平均查找和新增元素的时间均为 O（1），因此整个算法的时间复杂度为 O（n）;map 是基于红黑树实现的，查找时间复杂度为 log（n）; unordered_map 内部使用一个大的 hash_table , 期望查找时间为 O（1），相当于利用空间换时间。

# 解法三

  • 解题思路：双指针；如果不用哈希函数呢？第一种方法耗时间主要在两层循环上，怎么优化？对于三个数 a,b,c 和目标数值 t，分析如下：
  1. a+b=t, 则返回结果；
  2. a+b≠t, 则需要计算 a+c 和 b+c 与 t 的大小关系，但如果我们已经知道顺序关系 a<b<c 呢？

1） 如果 a+b<t；则我们希望增加 a 或 b 使得两数之和更接近 t，只需要计算 c+b 即可，因为 a+c<a+b<t；
2） 如果 a+b>t；则我们希望减小 a 或 b 使得两数之和更接近 t，只需要计算 a+c 即可，因为 c+b>a+b>t；

如果我们知道整个数组所有元素的排序关系，在选择元素时可以大大减少需要计算的二元组。
