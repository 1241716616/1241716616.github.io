  • 很多网站会检测某一段时间某个 IP 的访问次数，如果访问次数多的不想正常人，它会静止这个 IP 的访问。我们可以设置一些代理服务器，没隔一段时间换一个代理，就算 IP 被禁止，依旧可以换个 IP 继续爬。

# cookie

  • http 请求是无状态的，也就是说即使第一次和服务器连接后并且登录成功后，第二次请求服务器依然不知道当前请求是哪个用户。cookie 的出现是为了解决这个问题，第一次登录后服务器返回一些数据（cookie ）给浏览器，然后浏览器保存在本地，当用户第二次请求的时候，就会自动的把上次请求存储的 cookie 数据自动的携带给服务器，服务器通过浏览器携带的数据就能判断当前用户是哪个了。

# response.text 和 response.content 的区别

  • response.content：是直接从网络上庙抓取的数据，没有经过任何解码。是一个 bytes 类型。在硬盘上和网络上传输的字符串都是 bytes 类型。
  • response.text：是 str 的数据类型，是 response.content 进行解码的字符串。解码需要制定一个编码方式，response 会根据自己的猜测来判断编码方式，所以有时候可能会猜测错误，就会导致解码产生乱码，应该使用 response.content.decode ("utf-8") 进行手动解码。

# 发送 POST 请求

  • 最基本的 post 请求可以使用 post 方法，传入 data 就是 post 请求 ，大部分网站都是 get 请求。如果返回的是 json 数据，就可以调用 response.json () 将 json 字符串转为字典或列表。
