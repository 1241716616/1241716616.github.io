# 对算法的改进

任何算法得到的匹配结果都含有相当多的错误匹配项。

# 交叉检查匹配项

  • 说明：有一个简单的方法可以验证得到的匹配项，即重新进行同一个匹配过程，但在第二次将第二幅图象的每个关键点逐个与第一幅图像的全部关键点进行比较。只有在两个方向都匹配了同一对关键点时，才认为是一个有效的匹配项。函数 BFMatcher 提供了一个选项来使用这个策略。把有关标志设置为 true，函数就会对匹配项进行双向的交叉检查。在 BFMatcher matcher (NORM_L2，true)（交叉检查匹配项）。

# 比率检验法

  • 说明：当交叉检查匹配项也不理想时，是因为场景中有很多相似的物体，一个关键点可以和多个关键点匹配。

(1) 为此我们需要为每个关键点找到两个最佳的匹配项，可以用 Descriptor Matcher 类的 knnMatch 方法实现这个功能，因为只需要两个最佳匹配项，所以指定 k = 2;
(2) 下一步是排除与第二匹配项非常接近的全部最佳匹配项。做法：循环遍历每个关键点匹配项，然后执行比率验证法，即计算排名第二的匹配项与排名第一的匹配项的差值之比（如果两个最佳匹配项相等，那么比率为 1）。比率较高的匹配项将作为模糊匹配项，从结果中被排除掉。
