# 处理图像的颜色

# 用 GrabCut 算法分割图像

  • GrabCut 算法比较复杂，计算量也比较大，但结果很精确，如果要从静态图像中提取前景物体（例如从图像中剪切一个物体，并粘贴到另一幅图像），最好采用 GrabCut 算法

  • 算法原理：首先把所有未标记的像素临时标为前景（CV::GC_PR_FGD）, 基于当前的分类情况，算法把像素划分为多个颜色相似的组（既 k 个背景组和 k 个前景组）。通过引入前景和背景像素之间的边缘，确定背景 / 前景的分割，这将通过一个优化过程来实现。在此过程中，将试图连接具有相似标记的像素，并且避免边缘出现在强度较为均匀的区域。

输入 / 输出的图像可以是以下四个值：

  • CV::GC_BGD 表示明确属于背景的像素（例如本例中矩形之外的像素）
  • CV::GC_FGD 表示明确属于前景的像素（本例中没有这种像素）
  • CV::GC_PR_BGD 表示可能属于背景的像素
  • CV::GC_PR_FGD 表示可能属于前景的像素（本例中矩形之内像素的初始值）

# 用色调、饱和度和亮度表示颜色

  • 用于检测特色颜色时 HSV 色彩空间很实用，如肤色检测、路标检测等。

基础概念：
（1）色调（H）表示主色，我们使用的颜色名称（绿色、黄色等）就对应不同的色调值。
（2）饱和度（S）表示颜色的鲜艳程度，柔和的颜色饱和度较低彩虹的饱和度就很高。灰度颜色包含的 R、G、B 的成分是相等的，相当于一种极不饱和的颜色，所以它的饱和度是 0 （饱和度是一个 0 - 1.0 的值）。对于 8 位图像，饱和度被
调节成 0 - 255 的值，并且作为灰度图像显示的时候，较亮的区域对应的颜色具有较高的饱和度。
（3）亮度（V）是一个主观的属性，表示某种颜色的光亮程度。色调通常用 0 - 360 的角度来表示，其中红色是 0 度，对于 8 位图像，OpenCV 把亮度除以 2，以适合单字节的存储范围。

# image.copyTo () 的理解

  • 有两种形式：
    （1）image.copyTo (imageROI) ；作用是把 image 的内容粘贴到 imageROI
    （2）image.copyTo (imageROI,mask)； 作用是把 mask 和 image 重叠以后把 mask 中的像素值为 0 （黑）的点对应的 image 中的点变为透明，而保留其它点。

# threshold

函数原型：

double threshold(const Mat &srcImg, Mat &dstImg, double thresh,double maxVal,int type);

  • srcImg：原图像
  • dstImg：结果图像
  • thresh：当前阈值
  • maxVal：最大阈值
  • type：阈值类型

两种常用的阈值类型：
（1）THRESH BINARY：二进制阈值，。在运用该阈值类型的时候，先要选定一个特定的阈值量，比如：125，这样，
新的阈值产生规则可以解释为大于 125 的像素点的灰度值设定为最大值 255，灰度值小于 125 的像素点的灰度值设定为 0。(value>threshold?255:0)
（2） THRESH BINARY INV：反二进制阈值。设定一个初始阈值如 125，则大于 125 的设定为 0，而小于该阈值的设定为 255。 (value>threshold?0:255)
