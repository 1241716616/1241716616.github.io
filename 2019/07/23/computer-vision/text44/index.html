# Opencv 训练自己的物体分类器

说明： OpenCV 自带的的 opencv_traincascaded 分类器是 AdaBoost 级联分类器（前面有写过 AdaBoost 算法）。所谓的级联分类器，就是将若干的简单的分类器（可以理解为一般普通分类器）依次串联起来，最终的检测分类结果，要依次通过所有的分量分类器才能算是一个有效的检测分类结果，否则，就认为当前检测区域内没有我们需要找的目标。

# 收集训练样本

训练样本包括正样本和负样本；正样本，通俗点说就是图片中你需要的目标，而负样本的图片只要其中不含有目标就可以了，但需要说明，负样本也并非随便选取的；例如，你需要检测的目标是汽车，那么正样本就应该是仅仅含有汽车的图片，而负样本显然不能包含天空的、海洋的、风景的图片。因为你最终训练分类器的目的是检测汽车，而汽车应该出现在马路上，也就是说，分类器最终检测的图片应该是那些包含马路、交通标志、建筑物、广告牌、汽车、摩托车、三轮车、行人等在内的图片。很明显，这里的负样本应该是包含马路、交通标志、建筑物、广告牌、摩托车、三轮车、行人等。

# 对所有的正样本进行尺寸归一化

收集到的正样本有很多的尺寸大小，如 200 x 300, 500 X 800 等，尺寸归一化的目的是把所有的图片都缩放到同一大小。比如，都缩放到 24 x 24 ,50 x 60 等，可以是非 1：1 的长宽比，具体看你目标对象的尺寸而定。可以再在训练的时候在对应的参数设置，也可以在训练之前把图片统一处理准备好，文章后面会写一个小例子把文件夹里的全部图片缩放到指定的大小尺寸。

# 创建正样本 vec 文件

由于 LBP HAAR 训练的时候需要输入的正样本是 vec 文件，所以需要使用 createsample 程序来将正样本转换为 vec 文件。

CMD 进入 Windows 命令处理程序，进入你安装的 OpenCV 的目录找到 \bin 目录下，例如我的是如下图，我们会用到 opencv_createsamplesd.exe 程序来生成 vec 文件。OpenCV3.1 版本的是 opencv_createsamples.exe ，其实是同一个东西。

进入到 \bin 目录后 ，执行命令 opencv_createsamplesd 可查看需要传入的一些参数。

opencv_createsamplesd 参数说明：

  • -info <collection_file_name>：样本说明文件。

  • -img <image_file_name>：源目标图片。

  • -vec <vec_file_name>：训练好的正样本的输出文件名字。

  • -bg <background_file_name>：背景描述文件。

  • -num <number_of_samples>： 要产生的正样本数量，和正样本图片数目相同，其实就是样本描述文件第二列的所有数字累加。

  • -bgcolor <background_color>：背景色（假设当前图片为灰度图）。背景色制定了透明色，对于压缩图片，颜色方差量由 bgthresh 参数来指定，则在 bgcolor - bgthresh 和 bgcolor + bgthreshold 中间的像素被认为是透明的。

  • -inv：如果指定，颜色会反色

  • -randinv：如果指定，颜色

  • -maxidev <max_intensity_deviation>：背景色最大的偏离度

  • -maxxangle <max_x_rotation_angle> -maxyangle < max_y_rotation_angle> -maxzangle < max_z_rotation_angle > ：最大旋转角度，以弧度为单位。

  • show ：如果指定，每个样本都会被显示出来，按下 “Esc” 会关闭这一开关，即不显示样本图片，而创建过程继续。

  • -w <sample_width = 24> -h < sample_height = 24 > ：指明想让样本缩放到什么尺寸，如果你在这里设置了，就不用在前面提前处理正样本归一化这一步了，因为这个参数帮你同一缩放了，它也是输出样本的宽度和高度（以像素为单位）。

# 进行样本训练

该步骤调用 OpenCV \bin 目录下的 opencv_traincascaded.exe 程序。同样和上面生成的 vec 文件一样进入 \bin 目录（如果你已经退出），执行 opencv_traincascaded 查看需要传入的参数。

opencv_traincascaded 参数说明：

  • -data <cascade_dir_name>： 指定生成的文件目录。

  • -vec <vec_file_name>：vec 文件名，即正样本文件名（由 opencv_createsamplesd 程序生成的）。

  • -bg <background_file_name>：负样本描述文件名称。

  • -numPos <number_of_positive_samples> -numNeg < number_of_negative_samples >：用来训练每一个分类器阶段的正 / 负样本数目。一般设置负样本的数目为正样本的 1~3 倍。但是具体训练的时候 numPos 的值不能设置大于或等于正样本的数目，why？而 numNeg 可以设置大于负样本的数目，它会根据你设置的数目把负样本的图片自动裁切。

  • -numStages <number_of_stages>：训练的级联分类器层数。

  • -nsplits<number_of_stages>：决定用于阶段分类器的弱分类器。如果是 1，则一个简单的 stump classifier 被使用；如果是 2 或者更多，则带有 number_of_splits 个内部节点的 CART 分类器被使用；默认值为 2。

  • -minHitRate <min_hit_rate>：每个阶段分类器的最小命中率，即训练目标准确度。

  • -maxFalseAlarmRate <max_false_alarm_rate>：每个阶段的最大错误报警率。默认每一层训练到这个值小于 0.5 时训练结束，进入下一层训练。

  • -mem ：程序可使用的内存，这个设置为 256 即可，实际运行时根本就不怎么耗内
    存，以 MB 为单位。

  • -mode <BASIC (default) | CORE | ALL ：选择用来训练的 haar 特征集的种类，basic 仅仅使用垂直特征，all 使用垂直和 45 度角旋转特征。

  • -featureType <{HAAR (default), LBP, HOG} > ：选择训练所用特征，可以选择 HAAR、LBP、HOG , 但是 HOG 会报错，why ？不支持？HAAR 训练速度会慢一些，但精度会准确一点， LBP 速度较快。

  • -w <sampleWidth> -h < sampleHeight > ：训练样本的尺寸，必须和训练样本（vec 文件）创建的尺寸相同。

# 生成 xml 文件

当上一步程序训练结束以后，训练好的级联分类器将存储在文件 cascade.xml 文件中，这个文件位于 -data 指定的目录中。这个目录中的其它文件是训练的中间结果数据，当训练程序被中断后，再重新运行程序将读入之前的训练结果，而不需要从头重新训练，训练结束后就可以选择删除这些中间文件了。
